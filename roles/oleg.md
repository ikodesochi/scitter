**Задача 1: Нарисовать блок-схему AI кота**

Возьми лист бумаги или открой draw.io. Нарисуй блок-схему которая показывает как думает кот.

В центре схемы — 4 состояния кота соединённые стрелками:

ПАТРУЛЬ — кот ходит по маршруту. Из этого состояния есть стрелка в ПОГОНЮ с надписью «видит мышь».

ПОГОНЯ — кот бежит за мышью с увеличенной скоростью. Две стрелки: одна в ПОИСК с надписью «потерял из виду», другая на КОНЕЦ ИГРЫ с надписью «догнал».

ПОИСК — кот ходит кругами и ищет. Две стрелки: одна обратно в ПОГОНЮ с надписью «снова видит», другая в ПАТРУЛЬ с надписью «прошло 5 секунд».

ЗАМОРОЗКА — кот стоит на месте (собака помогла). Одна стрелка в ПАТРУЛЬ с надписью «прошло 3 секунды».

Сфоткай или сохрани схему в папку `docs/logic/cat-ai.png`.

Нарисуй вторую блок-схему для собаки. У неё 3 состояния:

БЛУЖДАНИЕ — собака ходит случайно по карте. Стрелка в ЗАЩИТУ с надписью «кот рядом с мышью и сытость > 20». Стрелка в СОН с надписью «сытость < 20».

ЗАЩИТА — собака бежит к коту и прогоняет его. Стрелка обратно в БЛУЖДАНИЕ с надписью «кот убежал». Тратит сытость.

СОН — собака не двигается. Стрелка в БЛУЖДАНИЕ с надписью «сытость > 30» (проснулась).

**Задача 3: Написать агрессию кота**

Открой файл `entities/cat.js`. Найди класс Cat и его метод `update()`.

В конструктор добавь новые переменные:
- `this.aggression = 0` — текущий уровень агрессии от 0 до 10
- `this.aggressionMax = 10` — максимум
- `this.aggressionDecay = 0.005` — скорость угасания когда мышь далеко

В метод `update()` после расчёта расстояния до мыши добавь:

Если расстояние до мыши меньше 100 пикселей:
- Увеличить агрессию на 0.02 за кадр
- Проверить чтобы не превысила максимум

Если расстояние больше 100 пикселей:
- Уменьшить агрессию на 0.005 за кадр
- Проверить чтобы не упала ниже нуля

Если агрессия больше 5:
- Увеличить зрение кота до 200 пикселей (вместо 150)
- Увеличить скорость до baseSpeed × 1.8

Если агрессия больше 8:
- Кот в ярости: ещё быстрее, зрение ещё дальше
- Показывать статус "!!!" над котом

**Задача 4: Написать сон собаки**

Открой файл `entities/dog.js`. Найди класс Dog и метод `update()`.

В конструктор добавь переменную:
- `this.sleeping = false` — спит ли собака

В метод `update()` в начале (перед всей логикой движения) добавь проверку:

Если `this.fullness < 20` и собака ещё не спит:
- Включить `this.sleeping = true`
- Показать статус "zZ" над собакой на 3 секунды

Если `this.sleeping === true`:
- Проверить если `this.fullness > 30`:
  - Выключить сон: `this.sleeping = false`
  - Показать статус "!" над собакой на 1 секунду
- Если всё ещё спит:
  - Выйти из метода (собака не двигается и не защищает)

**Задача 5: Написать физику столкновений**

Создай файл `core/physics.js`. Здесь будет код который проверяет можно ли пройти вперёд.

Функция `isWalkable(x, y)`:

- Делит x и y на TILE_SIZE чтобы получить координаты тайла
- Смотрит в карте уровня что находится в этом тайле
- Если тайл = 0 или тайл = 2 (пусто или пол) — возвращает true
- Если тайл = 1, 3, 4, 5 (стена, вода, шипы, другое) — возвращает false

Функция `tryMove(character, dx, dy)`:

- Считает новые координаты: newX = character.x + dx, newY = character.y + dy
- Проверяет 4 угла персонажа через isWalkable:
  - Левый верх: (newX, newY)
  - Правый верх: (newX + character.w, newY)
  - Левый низ: (newX, newY + character.h)
  - Правый низ: (newX + character.w, newY + character.h)
- Если по X все углы проходимы — двигает по X
- Если по Y все углы проходимы — двигает по Y
- Возвращает true если подвинулся хотя бы по одной оси

Функция `applyOilSliding(character)`:

- Проверяет стоит ли персонаж на масляном тайле
- Если да и таймер ещё не запущен:
  - Сохраняет обычную скорость в `character.originalSpeed`
  - Запускает `character.oilTimer = 60` (1 секунда)
- Пока таймер > 0:
  - Скорость = обычная скорость × 2
  - Блокирует смену направления
  - Уменьшает таймер на 1
- Когда таймер = 0:
  - Возвращает обычную скорость
  - Снимает блокировку

**Задача 6: Привязать физику к персонажам**

Теперь все персонажи должны использовать твою функцию tryMove вместо своего движения.

В файле `entities/cat.js` в методах движения замени прямое изменение координат на вызов `tryMove(this, dx, dy)`.

### 💬 ПОДСКАЗКИ

Для блок-схем используй простые фигуры: прямоугольники для действий, ромбы для условий (if), стрелки для переходов. Не усложняй — схема должна быть понятна с первого взгляда.

Для проверки углов персонажа обязательно проверяй все 4 точки. Если проверить только центр — персонаж будет наполовину заходить в стену.

Для масла храни таймер в кадрах а не в секундах. При 60 FPS одна секунда = 60 кадров. Уменьшай таймер на 1 каждый вызов update.

Если персонаж застревает в стене — проверь что ты используешь правильные размеры (w и h) при проверке углов.
