Ты первым создал репозиторий в Gitea на своём ноутбуке. Настроил три главных файла — `index.html` (страница с холстом), `style.css` (чёрно-белая тема) и `game.js` (сердце игры — игровой цикл). Когда ученик делает Pull Request, ты проверяешь код, подсказываешь что исправить, и если всё хорошо — принимаешь. Твоя задача — чтобы к концу занятия игра запускалась и показывала новый функционал.

⚙️ **game.js** — мозг игры. Главный цикл `update()`, который 60 раз в секунду обновляет всех персонажей, проверяет столкновения и вызывает отрисовку. Здесь же все импорты модулей учеников.

🗓️ **SPRINTS.md** — план по спринтам: кто что делает и когда дедлайн.

### 📝 ПЛАН ЗАНЯТИЯ

**🚀 Начало (5 минут)**

Раздай детям их карточки. Покажи на проекторе что сегодня в плане. Напомни кто в каких файлах работает и к кому идти с вопросами. Включи Gitea и убедись что все могут зайти.

**💻 Работа (30 минут)**

Дети открывают свои файлы и работают по карточкам. Ты ходишь по классу, смотришь кто как справляется. Если кто-то застрял — подсказываешь направление но не пишешь код за него. Если несколько человек с одинаковой проблемой — останавливаешь класс на минутку и объясняешь всем сразу.

Если у кого-то всё получается быстро — дай дополнительное задание: улучшить код, добавить комментарии, помочь соседу.

**🔧 Сборка (15 минут)**

Дети пушат свои ветки в Gitea и создают Pull Request. Ты по очереди открываешь каждый PR, проверяешь код, подсказываешь что исправить. Если всё ок — мержишь в main. После всех мержей открываешь игру и показываешь на проекторе что изменилось. Дети видят свой код в действии.

**🎉 Конец (10 минут)**

Отмечаешь в чек-листах кто что сделал. Хвалишь каждого за конкретную вещь: «Олег, агрессия кота работает отлично!», «Дамир, фон уровня 3 очень атмосферный!». Говоришь что будем делать на следующем занятии.

### ⚙️ ТВОИ ФУНКЦИИ

🟢 **initGame()** — создаёт игру при загрузке страницы

Получает canvas из index.html, создаёт контекст ctx, загружает первый уровень через levelManager, запускает главный игровой цикл. Вызывается один раз после заставки.

🔄 **gameLoop()** — главный цикл, 60 кадров в секунду

Очищает экран, обрабатывает нажатые клавиши (processInput), обновляет всех персонажей — мышь, кота, собаку, таракана. Проверяет столкновения: собрала ли мышь сыр, догнал ли кот, активировалась ли лунка. Вызывает render() для отрисовки всего на экране. В конце запрашивает следующий кадр через requestAnimationFrame.

🔀 **mergePullRequest(branchName)** — принять код ученика

Проверяет что код не ломает игру, принимает Pull Request в Gitea, решает конфликты если два ученика меняли один файл.

🆘 **helpStudent(studentName, problem)** — помочь ученику

Открывает ветку ученика, находит ошибку, объясняет как исправить. Не пишет код за него — только направляет.

🔌 **checkAllImports()** — проверить подключение модулей

Убеждается что все импорты в game.js правильные. Если кто-то из учеников создал новый файл — добавляет импорт. Без правильных импортов игра не соберётся.

📊 **drawAdminInfo(ctx)** — показывает информацию для отладки

Рисует в углу экрана координаты мыши, текущий тайл, скорость, здоровье, включён ли noclip или godmode. Работает когда нажата клавиша `\`.

### 🖥️ КОМАНДЫ GIT ДЛЯ ТЕБЯ

Принять Pull Request от ученика:
```
git checkout main
git merge feature/имя-ученика
git push origin main
```

Посмотреть какие ветки есть у всех:
```
git branch -a
```

Собрать всё вместе в конце занятия:
```
git checkout main
git merge feature/ivan
git merge feature/murad
git merge feature/oleg
... и так все 12 учеников
git push origin main
```

Если возник конфликт — Git покажет в каком файле. Открываешь файл, ищешь маркеры `<<<<<<<` и `>>>>>>>`, выбираешь правильную версию, сохраняешь, делаешь коммит.
