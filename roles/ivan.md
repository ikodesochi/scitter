### 📝 ЧТО ТЫ ДЕЛАЕШЬ

Ты даёшь персонажам лицо. Сейчас кот, мышь и собака — это просто серые прямоугольники с точками вместо глаз. Ты превратишь их в настоящих пиксельных персонажей в стиле 8bit.

Ты нарисуешь каждому персонажу несколько кадров анимации — когда они стоят, бегут, пугаются или злятся. И напишешь код который загружает эти спрайты в игру и правильно их показывает. После твоей работы игра станет выглядеть как настоящая ретро-игра из 90-х.

**Задача 1: Нарисовать спрайты кота**

Открой Piskel и создай спрайты для кота размером 32x32 пикселя. Рисуй в стиле 8bit pixel art — крупные пиксели, ограниченная палитра.

🐱 **cat-idle.png** — кот стоит (1 кадр)

Кот стоит на четырёх лапах, хвост опущен. Основной цвет — тёмно-серый или чёрный. Глаза — белые с чёрными зрачками, смотрят вперёд. Уши торчат вверх треугольниками. Хвост прямой.

🐱 **cat-run.png** — кот бежит (2 кадра)

Первый кадр: лапы расставлены шире, передние вытянуты вперёд, задние отставлены назад. Хвост поднят вверх. Второй кадр: лапы собраны вместе под телом, хвост опущен. При быстром переключении кадров создаётся иллюзия бега.

🐱 **cat-angry.png** — кот злой (1 кадр)

Тот же кот но глаза красные и светятся. Шерсть вздыблена — нарисуй зигзаги по спине. Рот открыт — видно белые клыки. Хвост распушён и поднят трубой. Этот спрайт показывается когда агрессия кота высокая.

🐱 **cat-frozen.png** — кот заморожен (1 кадр)

Кот стоит но всё тело голубоватого оттенка. Вокруг него — белые точки как снежинки. Глаза закрыты — две тонкие линии. Над головой можно нарисовать маленькие льдинки. Этот спрайт когда собака заморозила кота.

Сохрани все 4 файла в папку `assets/sprites/`.

**Задача 4: Написать загрузчик спрайтов**

Создай файл `systems/spriteRenderer.js`. В нём будет объект `spriteCache` где ключ — имя спрайта, а значение — загруженная картинка.

Функция `loadSprite(name, path)`:

- Создаёт объект Image
- Задаёт ему src = path (путь к файлу спрайта)
- Сохраняет в spriteCache под именем name
- Возвращает картинку когда она загрузится

Функция `loadAllSprites()`:

- Загружает все 11 спрайтов:
  - 'cat-idle', 'cat-run', 'cat-angry', 'cat-frozen'
  - 'mouse-idle', 'mouse-run', 'mouse-scared', 'mouse-dead'
  - 'dog-idle', 'dog-attack', 'dog-sleep'

Функция `drawSprite(ctx, name, x, y, w, h)`:

- Достаёт спрайт из spriteCache по имени
- Если спрайт загружен — рисует его через `ctx.drawImage()` в координатах x, y размером w на h
- Если не загружен — рисует запасной прямоугольник серого цвета

Функция `drawAnimatedSprite(ctx, name, x, y, w, h, frame, totalFrames)`:

- То же самое но для анимаций из нескольких кадров
- Вычисляет ширину одного кадра = ширина картинки / totalFrames
- Рисует только один кадр — тот что под номером frame
- Кадры переключаются извне (в update персонажа)

**Задача 5: Привязать спрайты к персонажам**

Теперь нужно заменить прямоугольники на спрайты в трёх файлах.

В файле `entities/cat.js` найди метод `draw()`. Замени отрисовку прямоугольника и глаз на вызов:

```javascript
if (this.frozen) {
    drawSprite(ctx, 'cat-frozen', x, y, this.w, this.h);
} else if (this.aggression > 5) {
    drawSprite(ctx, 'cat-angry', x, y, this.w, this.h);
} else if (this.chasing) {
    drawAnimatedSprite(ctx, 'cat-run', x, y, this.w, this.h, this.animFrame, 2);
} else {
    drawSprite(ctx, 'cat-idle', x, y, this.w, this.h);
}
```

В файле `entities/mouse.js` найди метод `draw()`. Замени на:

```javascript
if (this.health <= 0) {
    drawSprite(ctx, 'mouse-dead', x, y, this.w, this.h);
} else if (this.fear > 80) {
    drawSprite(ctx, 'mouse-scared', x, y, this.w, this.h);
} else if (this.isMoving) {
    drawAnimatedSprite(ctx, 'mouse-run', x, y, this.w, this.h, this.animFrame, 2);
} else {
    drawSprite(ctx, 'mouse-idle', x, y, this.w, this.h);
}
```

В файле `entities/dog.js` найди метод `draw()`. Замени на:

```javascript
if (this.sleeping) {
    drawSprite(ctx, 'dog-sleep', x, y, this.w, this.h);
} else if (this.attacking) {
    drawAnimatedSprite(ctx, 'dog-attack', x, y, this.w, this.h, this.animFrame, 2);
} else {
    drawSprite(ctx, 'dog-idle', x, y, this.w, this.h);
}
```

Добавь счётчик кадров анимации в классы персонажей: `this.animFrame = 0`. В методе `update()` увеличивай его: `this.animFrame = (this.animFrame + 1) % 2`.

### 💬 ПОДСКАЗКИ

Для анимаций из нескольких кадров делай спрайт широким: для 2 кадров — ширина 64 пикселя (2×32), для 4 кадров — 128 пикселей (4×32). Код сам нарежет по кадрам.

Рисуй персонажей на прозрачном фоне. В Piskel включи прозрачность (чекбокс «Transparent»). Тогда спрайт будет красиво накладываться на фон и тайлы.

Проверь что размеры спрайтов совпадают с размерами персонажей в коде (w и h). Если спрайт больше или меньше — он будет растянут или сжат.

Для переключения кадров анимации используй простой счётчик: увеличивай animFrame на 1 каждые 10 кадров игры (не каждый кадр, иначе анимация будет слишком быстрой).
